Midterm Project Proposal: SimonKeys

SimonKeys is a game based off the electronic game of memory skill Simon, which involved a physical interface of four buttons (red, green, blue, and yellow) and pressing them in a specified order. The object of the game was to see how many times the player could repeat back the colors before they couldn’t memorize them any longer. Linked is a picture of the game, as well as a video of the game being played. Players were assisted in memorizing the color sequence by the auditory assist of each button having a preassigned tone.

My idea for SimonKeys is for this same sequence-based memorization with the assist of tones, except instead of having four colored-buttons, the keyboard will be used. There will be three levels of difficulty: Easy, Medium, and Hard. Easy will have four keys from which to choose, Medium will have 12 keys from which to choose, and Hard will have all 26 lettered keys from which to choose. Each key will have a tone.

The sequence of the game will go like this:

  1. Choose a level of difficult
  2. Sequence is played back (e.g. A, Y, U, K)
  3. User attempts to relay the sequence back
    • If they fail, the game starts over and they can choose a new level of difficulty
    • If they succeed, the game continues with one more key to memorize


The user can pause or restart at any time and choose a new level of difficulty. There will be a points system.

I’m choosing to pursue this project because it’s a simple game that tests a very basic skill, memorization, with a fun twist of sound. I think it will be pretty challenging to code, as I’ll have to use some sort of stack or other way of storing the sequence of characters inputted to confirm if the user got it right. It will also involve the tonal functionality of P5, as well as an element of randomization. I think there are also different features I can add on if I have more time or want to make it more exciting. I can make experiment with graphics and tones a lot. It should be fun.
